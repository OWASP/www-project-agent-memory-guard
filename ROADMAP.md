# Roadmap

This document outlines the planned development for OWASP Agent Memory Guard.
Community contributions are welcome for any item below.

## v0.3.0 — Q2 2026

| Feature | Description | Status |
|---------|-------------|--------|
| LlamaIndex adapter | First-class integration with LlamaIndex memory backends | Planned |
| CrewAI adapter | Security middleware for CrewAI agent framework | Planned |
| Redis backend | Persistent memory store using Redis | Planned |
| PostgreSQL backend | Persistent memory store using PostgreSQL | Planned |
| Prometheus metrics | Export security event counts and latency metrics | Planned |
| Policy hot-reload | Reload YAML policies without restarting the agent | Planned |

## v0.4.0 — Q3 2026

| Feature | Description | Status |
|---------|-------------|--------|
| ML anomaly detection | Learned baselines for detecting unusual memory patterns | Planned |
| Vector-store protection | Guard vector embeddings from poisoning | Planned |
| Real-time dashboard | Web UI for monitoring security events | Planned |
| GitHub Action scanner | CI action to scan agent projects for vulnerabilities | Planned |
| Benchmark suite | Performance overhead measurements across frameworks | Planned |

## v1.0.0 — Q4 2026

| Feature | Description | Status |
|---------|-------------|--------|
| Multi-agent security | Cross-agent memory isolation and trust boundaries | Planned |
| OWASP Lab promotion | Apply for Lab project status | Planned |
| Formal verification | Prove policy correctness properties | Research |
| Enterprise features | RBAC, audit trails, compliance reporting | Planned |

## How to Contribute

Pick any item above and open an issue to discuss your approach. See
[CONTRIBUTING.md](CONTRIBUTING.md) for development setup instructions.
